Everything for Github can be done in VSCode

  1. Initialize a folder to reflect your repository (probably have already done this).
    • Open a folder and/or a file
    • Go to the Source Control trident on the left and click into it
    • Click Initialize Repository

  2. A trident 'main' 'asterik' will now be on the bottom left.

  3. Your file (in this case, index.html) will have a green U to the right of it.
    • A green U signals an "Untracked" file
    • Untracked files are file changes that have not been added to the repository

  4. Track this file by covering by the U and pressing the + button.
    • Pressing + stages the code

  5. A staged file will now have an A next to it
    • The A means the file has been staged but that's the only thing that plus does
    • Don't click anything yet: at this point, the action is only being described

  6. Next, you'll be committing the staged changes.
    • The M next to the file signals this step
    • Enter your commit Message in the box below the Source Control column header
    • Press checkmark next to that Source Control header

  7. Changes have now been made and saved, but create a branch for these changes:
    • Shift + Cmd + P to open the Command Palette
    • Type "create branch"
    • Name the branch
    • Check the bottom left of VSCode to see if the change is reflected by the swap of 'main' with the 'name of the new branch'
    • Create changes in your file and watch the gutter colors: green for additions; blue for edits; red for deletions
    • Add / stage the commits (see # 4)

  8. Review changes before any commits
    • Click on your file index.html
    • Then click the ... in the top right of the program
    • Inline view displays consolidated changes

  9. After making changes/your review, you stage the changes then commit them
    • Click the + above file in Source Code and below the commmit Message (NOT next to the file itself)
    • Enter your commit message after pressing the +
    • By the way, you can edit here too, by changing the green lines

  10. You're still in your created changes branch
    • Click the trident main on the bottom left and scroll to 'main' from within the Command Palette
    • This swaps you back into the main branch

  11. Now, to bring those changes into your main branch:
    • Click the ... at the top right of Source Control column
    • Then Branch > then Merge Branch
    • Scroll to the Branch you've been changing
    • Yes to everything
    • Yes to 'public'

  12. Enter as a 'public' repository
